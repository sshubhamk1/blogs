# Maximizing Network Reliability: Solving Network Recovery Pathways

## Problem Statement

In a network represented as a graph, we need to find a path from a starting node `0` to a destination node `n-1` that maximizes a specific metric: the **minimum edge weight** along that path (the "resultant"). However, there is a constraint: the total sum of the edge weights along the chosen path must not exceed a given threshold `k`. Additionally, we can only traverse through nodes that are marked as `online`.

**Goal:** Find the maximum possible "minimum edge weight" of a path from `0` to `n-1` such that the total weight sum $\leq k$ and all nodes in the path are `online`.

[3620. Network Recovery Pathways](https://leetcode.com/problems/network-recovery-pathways)

---

## The Challenge

At first glance, this looks like a shortest path problem. However, we aren't just looking for the shortest path; we are looking for a path that satisfies a "bottleneck" condition (maximizing the minimum weight) while adhering to a "budget" constraint (total weight $\leq k$).

A naive approach would be to explore all possible paths from `0` to `n-1`. However, since the number of paths in a graph can grow exponentially, a simple Depth First Search (DFS) or Breadth First Search (BFS) to find all paths will lead to a **Time Limit Exceeded (TLE)** error.

## The Optimal Approach: Binary Search on the Answer + Dijkstra

The key insight here is **monotonicity**. 

If we can find a path where every edge has a weight of at least $X$ and the total sum is $\le k$, then it is also possible to find such a path for any value less than $X$. This property allows us to use **Binary Search on the Answer**.

### Strategy

1.  **Binary Search Range:** The minimum possible weight is `0` and the maximum possible weight is the maximum edge weight in the graph (capped by `k`).
2.  **The Decision Function (`is_possible`):** For a given `target` weight, can we reach node `n-1` from node `0` using only edges with `weight >= target` such that the total path weight is $\le k$?
3.  **Shortest Path Verification:** To efficiently check the decision function, we use **Dijkstra's Algorithm**. For a fixed `target`, we only consider edges that meet the requirement (`edge_weight >= target`) and nodes that are `online`. Dijkstra's will find the *minimum* total weight to reach the destination under these constraints.
4.  **Final Decision:** If Dijkstra's finds a shortest path $\le k$, then the `target` is feasible, and we try a higher weight. Otherwise, we decrease our target.

## Implementation

```python
import heapq
import collections
import math
from typing import List

class Solution:
    def findMaxPathScore(self, edges: List[List[int]], online: List[bool], k: int) -> int:
        """
        Finds the maximum possible minimum edge weight in a path from 0 to n-1
        subject to a total weight constraint k and online node availability.
        """
        n = len(online)
        
        # Corner Case: If start or end is offline, no path is possible
        if not online[0] or not online[n-1]:
            return -1
        
        # Build adjacency list
        # graph[u] = [(v, weight), ...]
        graph = collections.defaultdict(list)
        for u, v, weight in edges:
            graph[u].append((v, weight))

        def is_path_possible(min_allowed_weight: int) -> bool:
            """
            Uses Dijkstra's algorithm to find if a path exists from 0 to n-1
            where every edge >= min_allowed_weight and total_sum <= k.
            """
            # dist[i] stores the minimum weight sum to reach node i
            min_dist = [math.inf] * n
            min_dist[0] = 0
            
            # Priority queue stores (current_total_weight, current_node)
            pq = [(0, 0)]
            
            while pq:
                current_weight_sum, u = heapq.heappop(pq)
                
                # If we found a shorter path to this node already, skip
                if current_weight_sum > min_dist[u]:
                    continue
                
                # Optimization: if we reached the destination, return True
                if u == n - 1:
                    return current_weight_sum <= k
                
                for v, weight in graph[u]:
                    # Check conditions: 
                    # 1. Node v must be online
                    # 2. Edge weight must be >= our binary search target
                    if online[v] and weight >= min_allowed_weight:
                        new_dist = current_weight_sum + weight
                        # 3. The total path sum must not exceed k
                        if new_dist <= k and new_dist < min_dist[v]:
                            min_dist[v] = new_dist
                            heapq.heappush(pq, (new_dist, v))
                            
            return min_dist[n-1] <= k

        # Binary search for the maximum possible "minimum weight"
        left, right = 0, k
        max_min_weight = -1
        
        while left <= right:
            mid = (left + right) // 2
            if is_path_possible(mid):
                max_min_weight = mid  # This weight is achievable, try for a larger one
                left = mid + 1
            else:
                right = mid - 1
                
        return max_min_weight

```

## Complexity Analysis

### Time Complexity
*   **Binary Search:** The search space is the range of weights (up to $k$ or $10^9$), which takes $O(\log(\text{max\\_weight}))$.
*   **Dijkstra's Algorithm:** For each step in the binary search, we run Dijkstra's, which takes $O(E \log V)$, where $E$ is the number of edges and $V$ is the number of nodes.
*   **Total Complexity:** $O(\log(\text{max\\_weight}) \cdot E \log V)$. Given the constraints typical for LeetCode, this is highly efficient.

### Space Complexity
*   **Adjacency List:** $O(V + E)$ to store the graph.
*   **Dijkstra Arrays:** $O(V)$ for the distance array and priority queue.
*   **Total Complexity:** $O(V + E)$.

## Key Takeaway
This problem is a classic example of **"Binary Search on Answer."** Whenever a problem asks you to "maximize the minimum" or "minimize the maximum" of something, your first instinct should be to check if the problem exhibits monotonicity, which then allows you to use binary search to reduce a complex optimization problem into a simpler feasibility check.
