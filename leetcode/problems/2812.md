# [2812. Find the Safest Path in a Grid](https://leetcode.com/problems/find-the-safest-path-in-a-grid/)

## Problem Overview

In this problem, we are given an $n \times n$ grid containing *0*s and *1*s. The *1*s represent the locations of thieves. The "safeness factor" of a cell is defined as the distance to the nearest thief. Our goal is to find a path from the top-left corner $(0,0)$ to the bottom-right corner $(n-1, n-1)$ such that the **minimum safeness factor** of any cell along that path is **maximized**.

Essentially, we want to find a path that stays as far away from the thieves as possible.

---

## Approach

This problem can be broken down into two distinct subproblems that we can solve sequentially:

### 1. Calculating Distance to the Nearest Thief (Multi-Source BFS)
The first step is to determine the "safeness factor" for every single cell in the grid. Since we need the distance to the *nearest* thief, a **Multi-Source Breadth-First Search (BFS)** is the most efficient approach.

*   We initialize a queue with the coordinates of all thieves (cells with value `1`).
*   We perform a level-order BFS. For each level of the BFS, we increment a distance counter.
*   As we traverse, we update the grid (or a distance matrix) to store the shortest distance from each cell to any thief. By the end of this process, `grid[r][c]` will represent the distance to the closest thief.

### 2. Finding the Maximum Safeness Path (Modified Dijkstra)
Once we have a grid where every cell contains its distance to the nearest thief, we need to find a path from $(0,0)$ to $(n-1, n-1)$ that maximizes the minimum value encountered along the way. This is a variation of the "Widest Path Problem."

*   We use a **Priority Queue (Max-Heap)** to always explore the cell that maintains the highest possible "current safeness factor."
*   Since Python’s `PriorityQueue` is a min-heap, we store distances as negative values to simulate a max-heap.
*   We maintain a `res` variable to keep track of the minimum distance encountered on the path so far.
*   We use a `visited` set to ensure we don't process the same cell multiple times, optimizing the search.
*   The first time we reach the destination $(n-1, n-1)$, the value we have tracked will be our answer.

---

## Complexity Analysis

*   **Time Complexity:** $O(N^2 \log N)$
    *   The Multi-source BFS visits each cell once: $O(N^2)$.
    *   The modified Dijkstra uses a Priority Queue where each of the $N^2$ cells can be pushed/popped once: $O(N^2 \log N)$.
*   **Space Complexity:** $O(N^2)$
    *   We use a `visited` set and a priority queue that, in the worst case, can store a significant portion of the $N^2$ cells.

---

## Implementation

```python
from collections import deque
from queue import PriorityQueue
from typing import List

class Solution:
    def maximumSafenessFactor(self, grid: List[List[int]]) -> int:
        n = len(grid)
        # Step 1: Find all thief locations to start Multi-Source BFS
        thiefs = [(i, j) for i in range(n) for j in range(n) if grid[i][j] == 1]
        
        visited = set()
        for thief in thiefs:
            visited.add(thief)
            
        # Multi-source BFS to calculate distance to nearest thief for every cell
        distance_level = 0
        directions = [(1, 0), (-1, 0), (0, 1), (0, -1)]
        queue = deque(thiefs)
        
        while queue:
            # Process current level of BFS
            for _ in range(len(queue)):
                r, c = queue.popleft()
                # Assign the current distance level to the cell
                grid[r][c] = distance_level
                
                for dr, dc in directions:
                    nr, nc = r + dr, c + dc
                    
                    # Boundary check and visited check
                    if 0 <= nr < n and 0 <= nc < n and (nr, nc) not in visited:
                        visited.add((nr, nc))
                        queue.append((nr, nc))
            distance_level += 1

        # Step 2: Modified Dijkstra to find path with maximum minimum-distance
        # The starting safeness is the value at the starting cell
        start_safeness = grid[0][0]
        visited.clear()
        visited.add((0, 0))
        
        # Max-heap using PriorityQueue (storing negative values for max-heap behavior)
        pq = PriorityQueue()
        pq.put((-start_safeness, 0, 0))
        
        max_min_safeness = start_safeness
        
        while not pq.empty():
            # current_min_dist is the minimum safeness encountered on the path to (i, j)
            current_min_dist, i, j = pq.get()
            current_min_dist = -current_min_dist
            
            # Update the overall result with the bottleneck distance
            max_min_safeness = min(max_min_safeness, current_min_dist)
            
            # If we reached the destination, return the result
            if i == n - 1 and j == n - 1:
                return max_min_safeness
            
            for dr, dc in directions:
                r, c = i + dr, j + dc
                
                if 0 <= r < n and 0 <= c < n and (r, c) not in visited:
                    visited.add((r, c))
                    # We pass the minimum of the current path's safeness and the neighbor's safeness
                    pq.put((-grid[r][c], r, c))
                    
        # Fallback return (though logic dictates the target is always reached in a grid)
        return grid[n-1][n-1]

```
