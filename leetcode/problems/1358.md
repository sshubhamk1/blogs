# [1358. Number of Substrings Containing All Three Characters](https://leetcode.com/problems/number-of-substrings-containing-all-three-characters)

## Problem Overview

Given a string `s` consisting of characters from 'a', 'b', and 'c', the goal is to find the total number of substrings that contain at least one occurrence of all three characters ('a', 'b', and 'c').

For example, if the input string is `"abcabc"`, the valid substrings would include `"abc"`, `"abca"`, `"abcab"`, `"abcabc"`, `"bca"`, and so on. Finding these efficiently is key to passing within the time limits.

---

## Approach 1: Sliding Window (Using a Frequency Map)

The first approach uses a **Sliding Window (Two Pointers)** technique combined with a hash map (or dictionary) to keep track of the character frequencies within the current window.

### The Logic
The core idea is to expand a window by moving a `right` pointer (`i`) until the window contains at least one 'a', one 'b', and one 'c'. 

Once this condition is met, we know that the current substring—and **every substring that extends from the current `i` to the end of the string**—will also satisfy the condition. Therefore, if the window from `last` to `i` is valid, we can immediately add the count of all remaining possible substrings starting from the current `last` position. We then shrink the window from the `left` (using the `last` pointer) to see if a smaller valid window exists.

### Implementation

```python
from collections import defaultdict

class Solution:
    def numberOfSubstrings(self, s: str) -> int:
        total_count = 0
        n = len(s)
        char_counts = defaultdict(int)
        left_pointer = 0

        # Helper function to check if the current window is valid
        def is_valid() -> bool:
            return char_counts['a'] > 0 and char_counts['b'] > 0 and char_counts['c'] > 0

        for right_pointer in range(n):
            # Add the current character to our window
            char_counts[s[right_pointer]] += 1

            # While the current window contains all three characters
            while is_valid():
                # If the window [left_pointer...right_pointer] is valid,
                # then every substring from right_pointer to the end of the string
                # starting at left_pointer is also valid.
                # However, to simplify logic: all substrings from left_pointer to 
                # ANY index >= right_pointer are valid.
                # A more efficient way: all substrings starting at left_pointer 
                # and ending at index >= right_pointer are valid.
                total_count += (n - right_pointer)
                
                # Shrink the window from the left to find the next potential valid window
                char_counts[s[left_pointer]] -= 1
                left_pointer += 1
                
        return total_count
```

### Complexity Analysis
*   **Time Complexity:** $O(N)$, where $N$ is the length of the string. Each pointer (`left` and `right`) travels across the string at most once.
*   **Space Complexity:** $O(1)$, because our dictionary will only ever store a maximum of three keys ('a', 'b', and 'c').

## Approach 2: Optimized Sliding Window (Tracking Last Seen Indices)

While the first approach is efficient, we can refine it further. Instead of maintaining a full frequency map and using a `while` loop to shrink the window, we can simply keep track of the **last seen index** for each of the three characters.

### The Logic
As we iterate through the string, we update the last recorded position of 'a', 'b', or 'c'. At any given index `i`, the smallest of the last seen positions of 'a', 'b', and 'c' tells us how many valid substrings end at the current index. 

If `min_index = min(last_a, last_b, last_c)`, then any substring starting at index `0, 1, ..., min_index` and ending at index `i` will contain all three characters. This gives us `min_index + 1` valid substrings ending at the current index.

### Implementation

```python
class Solution:
    def numberOfSubstrings(self, s: str) -> int:
        total_substrings = 0
        # Initialize the last seen positions of a, b, and c to -1
        last_seen = {'a': -1, 'b': -1, 'c': -1}

        for i, char in enumerate(s):
            # Update the last seen index for the current character
            last_seen[char] = i
            
            # The smallest index among the last seen positions of 'a', 'b', and 'c'
            # represents the starting point of the shortest valid substring ending at i.
            min_last_idx = min(last_seen['a'], last_seen['b'], last_seen['c'])
            
            # If min_last_idx is -1, it means we haven't seen all three characters yet.
            # If it is >= 0, all substrings starting from index 0 up to min_last_idx
            # and ending at the current index i are valid.
            total_substrings += (min_last_idx + 1) if min_last_idx != -1 else 0
            
            # Note: A more concise way to write the logic:
            # total_substrings += max(0, min(last_seen.values()) + 1)
            # However, we used the if-condition for clarity.
            
        return total_substrings
```

### Complexity Analysis
*   **Time Complexity:** $O(N)$, where $N$ is the length of the string. We perform a single pass through the string.
*   **Space Complexity:** $O(1)$, as we only store three integer variables regardless of the input size.

## Summary

| Approach | Technique | Time Complexity | Space Complexity | Pros/Cons |
| :--- | :--- | :--- | :--- | :--- |
| **Approach 1** | Sliding Window (Map) | $O(N)$ | $O(1)$ | Intuitive, but slightly more overhead due to dictionary updates and nested loop. |
| **Approach 2** | Last Seen Indices | $O(N)$ | $O(1)$ | Highly optimized, minimal operations per iteration. |

For this problem, **Approach 2** is the most elegant and efficient way to handle the constraints.
