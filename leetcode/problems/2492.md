# [2492. Minimum Score of a Path Between Two Cities](https://leetcode.com/problems/minimum-score-of-a-path-between-two-cities)

In this post, we will dive into a fascinating graph problem from LeetCode: **[2492. Minimum Score of a Path Between Two Cities](https://leetcode.com/problems/minimum-score-of-a-path-between-two-cities)**. This problem tests your ability to navigate connected components and find the optimal value within a subgraph.

## The Problem Statement

You are given a graph where cities are nodes and roads are edges with specific weights. You need to find the minimum "score" of a path between city $1$ and city $n$. The score is defined as the minimum weight of any road on the path. 

**Key Insight:** Since the problem implies you can travel back and forth and visit any city within the connected component containing city 1, the task essentially simplifies to: **Find the minimum edge weight in the entire connected component that includes city 1.**

We will explore two efficient ways to solve this: using **Breadth-First Search (BFS)** and using the **Disjoint Set Union (DSU)** data structure.

---

## Approach 1: Exploring the Network with BFS

The first approach involves traversing the entire connected component starting from city 1. We want to visit every reachable city and keep track of the smallest road weight we encounter along the way.

### The Logic
We can use a **Breadth-First Search (BFS)** to traverse the graph. We use an adjacency list to store the connections and their weights. As we visit each city, we inspect all its connected roads and update our global minimum weight.

### Implementation

```python
import collections

class Solution:
    def minScore(self, n: int, roads: List[List[int]]) -> int:
        # Build an adjacency list: city -> list of (neighbor, road_weight)
        adj = collections.defaultdict(list)
        for u, v, weight in roads:
            adj[u].append((v, weight))
            adj[v].append((u, weight))
        
        # BFS initialization
        visited = {1}
        min_road_weight = float('inf')
        queue = collections.deque([1])
        
        while queue:
            current_city = queue.popleft()
            
            for neighbor, weight in adj[current_city]:
                # We update the minimum weight found across all reachable roads
                min_road_weight = min(min_road_weight, weight)
                
                # If we haven't visited this city yet, add it to the queue
                if neighbor not in visited:
                    visited.add(neighbor)
                    queue.append(neighbor)
                    
        return min_road_weight
```

### Complexity Analysis
* **Time Complexity:** $O(V + E)$, where $V$ is the number of cities and $E$ is the number of roads. We visit each node and each edge at most once.
* **Space Complexity:** $O(V + E)$ to store the adjacency list and the visited set.

## Approach 2: Grouping Cities with Disjoint Set Union (DSU)

A more advanced way to handle connectivity problems is using the **Disjoint Set Union (DSU)**, also known as a Union-Find data structure. This is particularly powerful when we want to manage and merge different components efficiently.

### The Logic
We treat each city as a separate set initially. As we iterate through the roads, we `union` the two cities connected by the road. While merging, we also track the minimum road weight within each component. By the end, the minimum weight associated with the component containing city 1 will be our answer.

### Implementation

```python
import math

class DisjointSet:
    def __init__(self, n):
        self.parent = list(range(n + 1))
        # min_weight[i] stores the minimum weight road in the component represented by root i
        self.min_weight = [math.inf] * (n + 1)

    def find(self, i):
        # Path compression for efficiency
        if self.parent[i] == i:
            return i
        self.parent[i] = self.find(self.parent[i])
        return self.parent[i]

    def union(self, i, j, weight):
        root_i = self.find(i)
        root_j = self.find(j)

        if root_i != root_j:
            # Standard Union: connect one root to another
            self.parent[root_i] = root_j
            # The new component's min weight is the minimum of:
            # 1. The min weight in component I
            # 2. The min weight in component J
            # 3. The weight of the current road connecting them
            self.min_weight[root_j] = min(self.min_weight[root_i], self.min_weight[root_j], weight)
        else:
            # If they are already in the same component, just update the component's minimum
            self.min_weight[root_i] = min(self.min_weight[root_i], weight)

class Solution:
    def minScore(self, n: int, roads: List[List[int]]) -> int:
        ds = DisjointSet(n)
        
        for u, v, weight in roads:
            ds.union(u, v, weight)
            
        # The answer is the minimum weight found in the component containing city 1
        root_of_city_1 = ds.find(1)
        return ds.min_weight[root_of_city_1]
```

### Complexity Analysis
* **Time Complexity:** $O(E \cdot \alpha(V))$, where $\alpha$ is the inverse Ackermann function (nearly constant). This is incredibly efficient for large-scale connectivity problems.
* **Space Complexity:** $O(V)$ to store the `parent` and `min_weight` arrays.

---

## Summary: Which one should you use?

| Feature | BFS Approach | DSU Approach |
| :--- | :--- | :--- |
| **Best Use Case** | Simple traversal of a known graph. | Dynamic connectivity or grouping elements. |
| **Complexity** | $O(V + E)$ | $O(E \cdot \alpha(V))$ |
| **Memory** | Higher (due to Adjacency List) | Lower (Array-based) |

If the graph is static and you just need to visit all nodes, **BFS** is straightforward and intuitive. However, if you are dealing with a stream of connections or need highly optimized grouping, **DSU** is the industry-standard tool for the job!
