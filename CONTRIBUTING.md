# Contributing to [Your Blog Name/Repository Name]

First off, thank you for considering contributing! It's people like you who make this repository a valuable resource for the community.

## How Can I Contribute?

### 1. Adding LeetCode Problems
We are building a comprehensive collection of LeetCode solutions. To add a new problem:
1.  **Search first:** Make sure the problem doesn't already exist in the `leetcode/` directory.
2.  **Follow the format:** Use the existing structure in the `leetcode/` directory as a template. Each problem should have:
    *   A clear Markdown file named after the problem.
    *   A concise description of the problem.
    *   The optimal solution (with complexity analysis).
    *   Code in one or more languages (e.g., Python, Java, C++).
3.  **Structure:** Ensure your contribution follows the directory structure: `leetcode/<category>/<problem_name>.md`.

### 2. Writing Tech Blogs
We welcome high-quality technical content! If you want to write a blog post:
1.  **Check existing topics:** Ensure your topic hasn't been covered in depth recently.
2.  **Content Quality:** Articles should be informative, well-structured, and easy to follow.
3.  **Markdown Format:** Use Markdown for all content.
4.  **Directory:** Place your post in the appropriate directory (e.g., `system-design/`, `programming/`, `python/`, etc.).

### 3. Fixing Bugs or Improving Documentation
*   **Bugs:** If you find an error in a code solution or a typo in a blog post, please open an issue using the "Bug Report" template.
*   **Docs:** Small typos or documentation improvements are always welcome via Pull Requests.

## Submission Process

### Step 1: Fork the Repository
Click the "Fork" button on GitHub to create a copy of this repository in your account.

### Step 2: Create a Branch
Create a new branch for your changes:
```bash
git checkout -b feature/my-new-leetcode-problem
```

### Step 3: Make Your Changes
Add your content or fixes following the guidelines above.

### Step 4: Commit Your Changes
Write clear and concise commit messages:
```bash
git commit -m "Add LeetCode problem: Two Sum (Python/Java)"
```

### Step 5: Push and Open a Pull Request
Push your branch to your fork:
```bash
git push origin feature/my-new-leetcode-problem
```
Then, go to the original repository on GitHub and click "Compare & pull request".

## Pull Request Template
When opening a Pull Request, please:
- Describe the changes you've made.
- Link any related issues.
- Ensure your code/content follows the project's style.
